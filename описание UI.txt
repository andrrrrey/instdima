РАЗДЕЛ «ПЛАН» - ДОРАБОТКА ВЕРХНЕЙ ЧАСТИ

Цель:
Пользователь сразу должен понимать:
1. Что требует его внимания.
2. Какие публикации запланированы.
3. Какое действие нужно выполнить следующим.

1. ШАПКА РАЗДЕЛА

Заголовок:
Контент-план

Справа одна основная кнопка:
[+ Публикация]

Кнопка доступна:
- Еве;
- контент-мейкеру при включённом праве «Создавать публикации».

Дмитрию кнопку не показывать.

Убрать из раздела «План» кнопки:
- [Вставить];
- [Добавить идею];
- отдельную кнопку [Черновики].

«Добавить идею» должно находиться только в разделе «Идеи».
«Черновик» должен быть статусом публикации.

2. ОСНОВНЫЕ ВКЛАДКИ

Для Евы:
[Контроль] [Все] [Календарь]

Для контент-мейкера:
[Мои задачи] [Все] [Календарь]

Для Дмитрия:
[Календарь] [Список]

При открытии раздела по умолчанию показывать:
- Еве - «Контроль»;
- контент-мейкеру -«Мои задачи»;
- Дмитрию - «Календарь».

3. ВКЛАДКА «КОНТРОЛЬ» - ТОЛЬКО ДЛЯ ЕВЫ

В верхней части показать кликабельные счётчики:

[Просрочено]
[Дедлайн сегодня]
[На согласовании]
[На правках]
[Готово к выходу]

При нажатии на счётчик открывать список задач с соответствующим фильтром.

Ниже вывести блок «Требуют внимания»:

- просроченные задачи;
- задачи с дедлайном сегодня;
- публикации на согласовании;
- публикации, возвращённые на правки;
- публикации без ответственного;
- публикации, которые скоро выходят, но ещё не готовы.

4. ВКЛАДКА «МОИ ЗАДАЧИ» - ДЛЯ КОНТЕНТ-МЕЙКЕРА

В верхней части показать:

[Просрочено]
[Сегодня]
[На этой неделе]
[На согласовании]
[На правках]

Показывать только задачи, назначенные этому пользователю.

Задачи группировать:

- Просрочено;
- Сегодня;
- Завтра;
- На этой неделе;
- Выполнено.

5. ВКЛАДКА «ВСЕ»

Показывать публикации списком.

Фильтры:

[Период]
[Формат]
[Статус]

Для Евы дополнительно:

[Ответственный]
[Дедлайн]
[Только просроченные]

При активных фильтрах показывать:

[Фильтры · 2]

Добавить действие:

[Сбросить фильтры]

Контент-мейкер видит вкладку «Все» только при включённом праве «Видеть общий контент-план».

Дмитрию внутренние фильтры, дедлайны и ответственных не показывать.

6. ВКЛАДКА «КАЛЕНДАРЬ»

Переключатели:

[Неделя] [Месяц] [Календарь]

На телефоне по умолчанию открывать «Неделя».

Добавить навигацию:

[←] Текущий период [→]
[Сегодня]

Нажатие на свободную дату:

- Еве и контент-мейкеру открывает форму новой публикации с уже выбранной датой;
- Дмитрию открывает список публикаций на эту дату.

7. КАРТОЧКА ПУБЛИКАЦИИ

Для всех показывать:

- превью;
- название;
- Пост / Reels;
- дату и время публикации;
- общий статус.

Еве дополнительно показывать:

- ответственного;
- дедлайн подготовки;
- статус дедлайна;
- количество комментариев;
- дату последнего изменения.

Контент-мейкеру дополнительно показывать:

- свой дедлайн;
- комментарии;
- следующее необходимое действие.

Дмитрию не показывать:

- ответственного;
- дедлайн;
- просрочку;
- комментарии;
- историю изменений;
- внутренние рабочие статусы.

8. КНОПКИ НА КАРТОЧКЕ

Для контент-мейкера показывать только одну кнопку в зависимости от статуса:

Черновик:
[Начать работу]

В работе:
[Продолжить]

На правках:
[Внести правки]

Готово к проверке:
[Отправить на согласование]

Готово к выходу:
[Отметить опубликованной]

Для Евы:

На согласовании:
[Согласовать]
[Вернуть на правки]

Для остальных статусов:
[Открыть]

Дополнительные действия убрать в меню [•••]:

- изменить дедлайн;
- изменить ответственного;
- перенести;
- дублировать;
- архивировать.

Для Дмитрия:

[Открыть]

9. СТАТУСЫ

Рабочие статусы:

- Черновик;
- В работе;
- На согласовании;
- На правках;
- Готово к выходу;
- Опубликовано;
- Отменено.

Статус дедлайна рассчитывать отдельно:

- В срок;
- Дедлайн сегодня;
- Просрочено;
- Выполнено в срок;
- Выполнено с просрочкой.

Дмитрий видит только:

- В работе;
- На согласовании;
- Готово к выходу;
- Опубликовано.

10. ОБЩИЕ ТРЕБОВАНИЯ

- В шапке раздела должна быть только одна главная кнопка: [+ Публикация].
- Состав вкладок, фильтров и кнопок зависит от роли и выданных прав.
- Недоступные функции полностью скрывать.
- Дедлайн подготовки и дата публикации должны быть разными полями.


- Нажатие на счётчик, статус или фильтр должно сразу открывать соответствующий список.
- После возврата на экран сохранять выбранную вкладку, период календаря и фильтры.
- На мобильном экране верхняя часть не должна занимать больше трети экрана.
